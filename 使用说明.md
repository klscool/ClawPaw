# ClawPaw 使用说明

**协议参考**：WebSocket 握手与消息格式以 [OpenClaw Gateway Protocol](https://docs.openclaw.ai/gateway/protocol) 为准（connect.challenge → connect、device 身份与签名、req/res 帧）。

---

## 一、App 界面

- **初始化步骤**：按「初始化」依次检查并引导：无障碍服务（必选）、输入法「ClawPaw 输入法」（建议）、OpenClaw 连接（可选）。各项可点击刷新状态；未完成项会打 ✗，可选未连接显示 ?。
- **服务与网络**：展示当前 HTTP 端口（默认 8765）与本机 IP，供主机通过 Tailscale 等访问。
- **右上角菜单**：修改 Gateway 地址、设置/修改 Gateway Token、清除密钥对（重新配对）、调试 Toast 开/关、SSH / 隧道配置。
- **输入法**：执行 `input_text` 前需在系统中启用本应用键盘；菜单或输入法卡片可进入系统输入法设置。
- **调试 Toast**：开启后，每次收到命令（WebSocket / HTTP / ADB）会在前台弹 Toast、在后台更新「命令」通知，便于在其它应用时确认收到的指令。

---

## 二、控制方式概览

| 方式 | 说明 |
|------|------|
| **A. WebSocket（OpenClaw node）** | App 连接 Gateway，由 OpenClaw 下发命令，应用执行并回结果。 |
| **B. HTTP** | 手机本地 HTTP 服务（端口 8765），主机通过 Tailscale 等直连：GET 布局/截图，POST 执行命令。 |
| **C. ADB** | 通过 ADB 广播下发单条命令，用于开发/验收原子能力。 |

**仅能 SSH + WebSocket 时**：手机与主机不互通则无法直连 8765，只能让手机 **WS 连到 Gateway**，在主机用 **OpenClaw node 命令**（或 Control UI）控制，即**仅方式 A**。

## 三、命令能力总览

所有命令在 **OpenClaw node**、**HTTP**、**ADB** 三种方式下均可使用（方式 A/B 需先连上或能访问到手机）。通过 OpenClaw 控制时，有原生 action 的优先用原生（device_status、device_info、location_get、notifications_list、camera_snap、photos_latest），其余用 invoke。以下按**类别**分表列出。

### 3.1 无障碍 / 界面

需开启无障碍服务。执行后默认等待 2 秒并返回当前布局；传 `return_layout_after: false` 可不等待、不返回布局。

| 命令 | 说明 |
|------|------|
| `get_layout` | 获取当前界面布局（XML）。 |
| `screenshot` | 截屏，返回 Base64 或路径。 |
| `click` | 点击坐标 `x,y`。 |
| `input_text` | 在坐标处点击并用本应用输入法输入（需已启用）。 |
| `input_text_direct` | 无障碍直接设文字（不依赖输入法）。 |
| `swipe` | 滑动 `start_x,start_y → end_x,end_y`。 |
| `long_press` | 长按坐标 `x,y`，700ms。 |
| `two_finger_swipe_same` | 两指同向张开放大（如放大地图），部分机型可能不生效。 |
| `two_finger_swipe_opposite` | 两指反向放大/缩小：并拢=缩小、张开=放大。 |
| `back` | 返回键。 |
| `open_schema` | 按 schema/URI 或包名打开应用。 |

### 3.2 设备状态（不依赖无障碍）

| 命令 | 说明 |
|------|------|
| `location_get` | 最后已知定位（需定位权限），返回 `lat/lon` 或 `error`。 |
| `get_wifi_name` | 当前 WiFi SSID。 |
| `get_screen_state` | 屏幕亮/灭，返回 `on` / `off`。 |
| `get_state` | 一次返回：定位 + WiFi + 屏幕亮灭 + 电量 + 屏幕分辨率 screen_width/screen_height（JSON）。 |
| `device_status` | 与 `get_state` 等价，返回带 `ok: true` 的状态 JSON。 |
| `device_info` | 设备信息：model、manufacturer、androidVersion、displayName 等。 |
| `device.health` | 设备健康：memory、battery 详情、power、system（securityPatchLevel）；**仅 Node**。 |
| `device.permissions` | 各权限 status（granted/denied）与 promptable；**仅 Node**。 |

### 3.3 硬件（不依赖无障碍）

| 命令 | 说明 |
|------|------|
| `vibrate` | 震动，可选 `duration_ms`（默认 200）。 |
| `camera_rear` / `camera_front` | 后置/前置拍照，异步保存，返回已启动。 |
| `camera_snap` | 拍照，参数 `facing`：0 后置、1 前置。 |
| `screen_on` | 点亮/唤醒屏幕。 |

### 3.4 通知（需通知监听权限）

| 命令 | 说明 |
|------|------|
| `notifications_list` | 获取当前通知列表（含 `key` 供 actions 使用）。 |
| `notifications.actions` | 操作通知：`action=dismiss`、`key=` 来自 list；**仅 Node**。open/reply 未实现。 |
| `notification.show` / `notifications.push` / `system.notify` | 推送本地通知，参数 `title`、`text` 或 `body`（与官方一致）。 |

### 3.5 联系人（需 READ_CONTACTS）

| 命令 | 说明 |
|------|------|
| `contacts.list` / `contacts.search` | 联系人列表，可选 `limit`（默认 500）；与官方声明对齐。 |

### 3.6 照片（需存储 / READ_MEDIA_IMAGES）

| 命令 | 说明 |
|------|------|
| `photos_latest` | 最近照片列表（id、displayName、dateAdded、size），可选 `limit`（默认 50）。 |

### 3.7 日历（需 READ_CALENDAR）

| 命令 | 说明 |
|------|------|
| `calendar.list` / `calendar.events` | 日历事件列表，可选 `limit`（默认 100）。 |

### 3.8 音量

| 命令 | 说明 |
|------|------|
| `volume.get` | 媒体/铃声音量与最大值。 |
| `volume.set` | 设置音量，参数 `stream`（media/ring）、`volume`。 |

### 3.9 文件（需存储权限）

| 命令 | 说明 |
|------|------|
| `file.read_text` | 读取文件为 UTF-8 文本，参数 `path`。 |
| `file.read_base64` | 读取文件为 Base64，参数 `path`。 |

### 3.10 传感器与运动（步数需 ACTIVITY_RECOGNITION）

| 命令 | 说明 |
|------|------|
| `sensors.steps` | 步数（steps、available）。 |
| `sensors.light` | 光照 lux。 |
| `sensors.info` | 传感器列表摘要。 |
| `motion.pedometer` | 计步：参数可选 `startISO`/`endISO`，返回 startISO、endISO、steps；**仅 Node**。 |
| `motion.activity` | 活动识别：返回 activities[]（isStationary、isWalking 等）；**仅 Node**。 |

### 3.11 蓝牙（需 BLUETOOTH_CONNECT）

| 命令 | 说明 |
|------|------|
| `bluetooth.list` | 已配对设备列表（name、address）。 |

### 3.12 WiFi

| 命令 | 说明 |
|------|------|
| `wifi.info` | WiFi 状态（enabled、ssid、bssid、rssi、ipAddress）。 |
| `wifi.list` | 附近 WiFi 列表（扫描结果），会先触发一次扫描再返回；每项含 ssid、bssid、level（信号强度）、frequency、capabilities。Android 6+ 需定位权限且定位开启，否则列表可能为空。 |
| `wifi.enable` | 开/关 WiFi，参数 `enabled`。 |

### 3.13 短信（需 READ_SMS、SEND_SMS）

| 命令 | 说明 |
|------|------|
| `sms.list` | 收件箱最近短信，返回 address、body、date、type；可选 `limit`（默认 50）。 |
| `sms.send` | 发送短信，参数 `address`（或 `to`）、`body`（或 `text`）。 |

### 3.14 电话（需 CALL_PHONE）

| 命令 | 说明 |
|------|------|
| `phone.dial` | 打开拨号界面并预填号码，参数 `number`（或 `phone`）；不直接拨出。 |
| `phone.call` | 直接拨打电话，参数 `number`（或 `phone`）。 |

### 3.15 铃声与勿扰

| 命令 | 说明 |
|------|------|
| `ringer.get` | 当前铃声模式：normal / silent / vibrate。 |
| `ringer.set` | 设置铃声模式，参数 `mode`：`normal` \| `silent` \| `vibrate`。修改需通知策略访问权限（部分机型需用户到系统设置授权）。 |
| `dnd.get` | 勿扰状态：enabled、filter（all / priority / alarms / none）。 |
| `dnd.set` | 设置勿扰，参数 `enabled`：true 为仅闹钟，false 为关闭勿扰；需通知策略访问权限。 |

**Gateway 认证**：Node 设置中 Token 与密码二选一，连接时优先使用 Token；部分网关支持密码认证。

## 四、使用示例

三种方式统一约定：**无参数**命令传 `{}` 或不写键；**有参数**的传对应键值；**带文字**的需传 `text`/`title`/`body` 等。下面每种方式各给三类示例，再列「命令/action + 主要参数」表。

---

### 4.1 OpenClaw Node 调用（方式 A）

主机执行 `openclaw nodes status` 得到节点 id，将 `NODE_ID` 替换后执行。

**格式**：`openclaw nodes invoke --node NODE_ID --command <命令> --params '<JSON>'`

**示例（无参数）**：

```bash
openclaw nodes invoke --node NODE_ID --command back --params '{}'
openclaw nodes invoke --node NODE_ID --command get_state --params '{}'
```

**示例（有参数，如坐标/数字）**：

```bash
openclaw nodes invoke --node NODE_ID --command click --params '{"x":500,"y":1000}'
openclaw nodes invoke --node NODE_ID --command volume.set --params '{"stream":"media","volume":5}'
```

**示例（带文字）**：

```bash
openclaw nodes invoke --node NODE_ID --command input_text_direct --params '{"x":300,"y":800,"text":"你好"}'
openclaw nodes invoke --node NODE_ID --command system.notify --params '{"title":"标题","body":"内容"}'
```

**原生 Action**（节点支持以下名称；**命令行**一律用 `nodes invoke --command <命令名>`）：`device_status`、`device_info`、`location_get`、`notifications_list`、`camera_snap`、`photos_latest`。参数见 [OpenClaw 文档](https://docs.openclaw.ai)。  
Node 端上述 6 个命令**同时接受点号写法**（`device.status`、`device.info`、`location.get`、`notifications.list`、`camera.snap`、`photos.latest`），与 OpenClaw 下发的 action 名兼容。

**须用 Invoke 的命令与主要参数**（与命令一一对应）：

| 命令 | 主要参数 |
|------|----------|
| `get_layout` | 无 |
| `screenshot` | 无 |
| `back` | 无 |
| `location_get`、`get_wifi_name`、`get_screen_state`、`get_state`、`device_status`、`device_info` | 无 |
| `camera_rear`、`camera_front`、`camera_snap`、`screen_on` | 无（camera_snap 可选 `facing`：0 后置、1 前置） |
| `volume.get`、`sensors.steps`、`sensors.light`、`bluetooth.list`、`wifi.info`、`wifi.list` | 无 |
| `click`、`long_press` | `x`, `y`；可选 `return_layout_after` |
| `input_text`、`input_text_direct` | `x`, `y`, `text` |
| `swipe` | `start_x`, `start_y`, `end_x`, `end_y` |
| `two_finger_swipe_same`、`two_finger_swipe_opposite` | `start_x`, `start_y`, `end_x`, `end_y` |
| `open_schema` | `schema` 或 `uri`（包名或 URI） |
| `vibrate` | 可选 `duration_ms` |
| `system.notify`、`notification.show` | `title`，`body` 或 `text`（或 heading/subject、message/content） |
| `contacts.list`、`contacts.search`、`calendar.list`、`calendar.events`、`photos_latest`、`notifications_list` | 可选 `limit` |
| `volume.set` | `stream`（media/ring），`volume`（必填 0–15 或与系统一致） |
| `file.read_text`、`file.read_base64` | `path`（必填） |
| `wifi.enable` | `enabled` |
| `wifi.list` | 无 |
| `sms.list` | 可选 `limit` |
| `sms.send` | `address` 或 `to`，`body` 或 `text` |
| `phone.dial`、`phone.call` | `number` 或 `phone` |
| `ringer.get` | 无 |
| `ringer.set` | `mode`（normal / silent / vibrate） |
| `dnd.get` | 无 |
| `dnd.set` | `enabled` |
| **仅 Node**：`device.health`、`device.permissions` | 无 |
| **仅 Node**：`motion.pedometer` | 可选 `startISO`、`endISO`（ISO8601 时间） |
| **仅 Node**：`motion.activity` | 无 |
| **仅 Node**：`notifications.actions` | `action`（如 dismiss），`key`（来自 notifications_list） |

**节点声明**：连接 Gateway 时，App 会上报支持的 commands 列表（含 `contacts.list`、`calendar.list`、`sms.list`、`phone.dial`、`phone.call`、`ringer.set`、`dnd.set` 等）。若 OpenClaw 仍提示「节点不支持」某命令，请先**重连** Gateway 以刷新声明；若仍不行，可能是服务端对 Android 节点的允许列表限制，需在 OpenClaw 侧配置。

**SSH + WS**：手机 App 填主机地址（如 Tailscale IP），`ws://该地址:18789` 连接；主机执行上述命令即可。

### 4.2 HTTP（方式 B）

前提：手机与主机网络互通。将 `PHONE_IP` 换为手机 IP。POST 体为 JSON，必须含 `action`。

**示例（无参数）**：

```bash
curl -s http://PHONE_IP:8765/api/status
curl -s http://PHONE_IP:8765/api/layout
curl -s -X POST -H "Content-Type: application/json" -d '{"action":"back"}' http://PHONE_IP:8765/api/execute
```

**示例（有参数）**：

```bash
curl -s -X POST -H "Content-Type: application/json" \
  -d '{"action":"click","x":500,"y":1000}' http://PHONE_IP:8765/api/execute
```

**示例（带文字）**：

```bash
curl -s -X POST -H "Content-Type: application/json" \
  -d '{"action":"input_text_direct","x":300,"y":800,"text":"你好"}' http://PHONE_IP:8765/api/execute
```

**示例（WiFi 列表）**：

```bash
curl -s -X POST -H "Content-Type: application/json" -d '{"action":"wifi.list"}' http://PHONE_IP:8765/api/execute
```

成功响应形如 `{"success":true,"result":...}` 或带 `layout`（Base64）。

**action 与主要参数**（与 Node 命令对应，仅传参方式为 HTTP 的 `action` + 同名字段）：

| action | 主要参数 |
|--------|----------|
| `get_layout`、`screenshot`、`back` | 无 |
| `location_get`、`get_wifi_name`、`get_screen_state`、`get_state`、`device_status`、`device_info` | 无 |
| `camera_rear`、`camera_front`、`screen_on`、`volume.get`、`sensors.steps`、`sensors.light`、`bluetooth.list`、`wifi.info`、`wifi.list` | 无 |
| `click`、`long_press` | `x`, `y`；可选 `return_layout_after` |
| `input_text`、`input_text_direct` | `x`, `y`, `text` |
| `swipe` | `start_x`, `start_y`, `end_x`, `end_y` |

| `open_schema` | `schema` 或 `uri` |
| `vibrate` | 可选 `duration_ms` |
| `system.notify`、`notification.show`、`notifications.push` | `title`，`body` 或 `text` |
| `contacts.list`、`contacts.search`、`calendar.list`、`calendar.events`、`photos_latest`、`notifications_list` | 可选 `limit` |
| `volume.set` | `stream`（media/ring），`volume`（必填） |
| `file.read_text`、`file.read_base64` | `path`（必填） |

### 4.3 ADB（方式 C）

**格式**：必须带 `-n` 指定组件；`command` 为 JSON 字符串，需包含 `action`。

**示例（无参数）**：

```bash
adb shell am broadcast -n com.example.clawpaw/.receiver.CommandReceiver \
  -a com.example.clawpaw.EXECUTE_COMMAND --es command '{"action":"back"}'
```

**示例（有参数）**：

```bash
adb shell am broadcast -n com.example.clawpaw/.receiver.CommandReceiver \
  -a com.example.clawpaw.EXECUTE_COMMAND --es command '{"action":"click","x":500,"y":1000}'
```

**示例（带文字）**：

```bash
adb shell am broadcast -n com.example.clawpaw/.receiver.CommandReceiver \
  -a com.example.clawpaw.EXECUTE_COMMAND --es command '{"action":"input_text_direct","x":300,"y":800,"text":"你好"}'
```

Windows CMD 下 JSON 内双引号写为 `\"`。**action 与主要参数**见 **六、ADB 命令参考**。

## 五、HTTP 接口详情（方式 B）

服务监听 `0.0.0.0:8765`，应用启动后由前台 Service 保活。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/status` | GET | 健康检查。 |
| `/api/layout` | GET | 当前界面布局，`layout` 为 Base64。 |
| `/api/screenshot` | GET | 截屏，`screenshot` 为 Base64（或 `saved_to_gallery`）。 |
| `/api/execute` | POST | 执行单条命令，body 见下。 |
| `/api/task` | POST | 预留。 |

**POST /api/execute**：body 为 JSON，必须含 `action`。可选 `return_layout_after`（默认 `true`：执行后等 2 秒并返回布局）。action 与 **三、命令能力总览** 及 **4.1 OpenClaw Node 调用** 一一对应，仅传参方式为 HTTP 的 `action` + 同名字段。

**无障碍 / 界面**：`click`（x, y）、`input_text`（x, y, text）、`input_text_direct`、`swipe`、`long_press`、`two_finger_swipe_same`、`two_finger_swipe_opposite`、`back`、`open_schema`（schema/uri）、`get_layout`。

**设备状态**：`location_get`、`get_wifi_name`、`get_screen_state`、`get_state`、`device_status`、`device_info`。

**硬件**：`vibrate`（duration_ms）、`camera_rear`、`camera_front`、`camera_snap`（facing）、`screen_on`。

**通知**：`notifications_list`、`notifications.actions`（action, key）、`notification.show` / `notifications.push` / `system.notify`（title, text/body）。`device.health`、`device.permissions`、`motion.pedometer`、`motion.activity`、`notifications.actions` 仅 Node(WebSocket) 实现，HTTP/ADB 返回未实现提示。

**联系人 / 照片 / 日历**：`contacts.list` / `contacts.search`（limit）、`photos_latest`（limit）、`calendar.list` / `calendar.events`（limit）。

**音量 / 文件**：`volume.get`、`volume.set`（stream, volume）、`file.read_text`（path）、`file.read_base64`（path）。

**传感器 / 蓝牙 / WiFi**：`sensors.steps`、`sensors.light`、`sensors.info`、`bluetooth.list`、`wifi.info`、`wifi.list`、`wifi.enable`（enabled）。

**短信 / 电话**：`sms.list`（limit）、`sms.send`（address/to, body/text）、`phone.dial`、`phone.call`（number/phone）。

**铃声与勿扰**：`ringer.get`、`ringer.set`（mode：normal/silent/vibrate）、`dnd.get`、`dnd.set`（enabled）。

## 六、ADB 命令参考（方式 C）

**通用格式**（必须带 `-n`）：

```bash
adb shell am broadcast -n com.example.clawpaw/.receiver.CommandReceiver \
  -a com.example.clawpaw.EXECUTE_COMMAND --es command "<JSON>"
```

**约定**：状态与硬件类不依赖无障碍；其余为无障碍类，需已开启本应用无障碍服务。

**action 与主要参数**（与 Node/HTTP 命令对应，body 为 `command` 里的 JSON）：

| action | 主要参数 |
|--------|----------|
| `get_layout`、`screenshot`、`back` | 无 |
| `location_get`、`get_wifi_name`、`get_screen_state`、`get_state`、`device_status`、`device_info` | 无 |
| `camera_rear`、`camera_front`、`camera_snap`、`screen_on`、`volume.get`、`sensors.steps`、`sensors.light`、`bluetooth.list`、`wifi.info`、`wifi.list` | 无（camera_snap 可选 `facing`） |
| `click`、`long_press` | `x`, `y`；可选 `return_layout_after` |
| `input_text`、`input_text_direct` | `x`, `y`, `text` |
| `swipe` | `start_x`, `start_y`, `end_x`, `end_y` |
| `two_finger_swipe_same`、`two_finger_swipe_opposite` | `start_x`, `start_y`, `end_x`, `end_y` |
| `open_schema` | `schema` 或 `uri` |
| `vibrate` | 可选 `duration_ms` |
| `system.notify`、`notification.show` | `title`，`body` 或 `text` |
